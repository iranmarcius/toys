A API Toys foi construída para prover funcionalidades não existentes em outras APIs opensource e complementar as funcionalidades já existentes nestas.

Ela foi implementada à partir de uma série de outras APIs tais como Jakarta Commons, Hibernate, e outras, oferecendo uma gama de métodos utilitários para auxiliar na utilização dessas, o que significa que a Toys possui dependências com essas APIs. Essas dependências são indicadas na documentação das classes e pacotes.
